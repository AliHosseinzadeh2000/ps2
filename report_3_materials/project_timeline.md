## فاز ۱ — معماری و یکپارچه‌سازی صرافی‌ها

**بازه زمانی:** فوریه ۲۰۲۵  
**گزارش:** گزارش شماره ۱ (تحویل داده شد — فوریه ۲۵، ۲۰۲۵)  
**وضعیت:** ✅ کامل

### آنچه ساخته شد

| کامپوننت | توضیح |
|----------|-------|
| `ExchangeInterface` | رابط پایه برای تمام صرافی‌ها (Strategy Pattern) |
| `NobitexExchange` | صرافی نوبیتکس — احراز هویت Token |
| `WallexExchange` | صرافی والکس — احراز هویت API Key |
| `InvexExchange` | صرافی اینوکس — احراز هویت RSA |
| `KucoinExchange` | صرافی کوکوین — HMAC-SHA256 |
| `TabdealExchange` | صرافی تبدیل |
| `SymbolConverter` | تبدیل نماد بین فرمت‌های مختلف (IRT/USDT/IRR) |
| `ArbitrageEngine` | موتور شناسایی فرصت آربیتراژ |

### نقاط مهم تصمیم‌گیری
- انتخاب معماری async (asyncio + httpx) برای همزمانی
- مشخص کردن IRT، IRR، TMN به‌عنوان یک ارز (تومان)
- استفاده از Pattern Factory برای exchange‌ها

## فاز ۲ — موتور اصلی معامله‌گری

**بازه زمانی:** فوریه ۲۰۲۵ — دسامبر ۲۰۲۵  
**گزارش:** گزارش شماره ۲ (تحویل داده شد — دسامبر ۱۳، ۲۰۲۵)  
**وضعیت:** ✅ کامل

### آنچه ساخته شد

| کامپوننت | توضیح |
|----------|-------|
| `OrderExecutor` | اجرای همزمان دو سفارش خرید/فروش با asyncio.gather |
| `PriceStream` | polling مداوم orderbook از صرافی‌ها |
| `CircuitBreakers` | ۳ نوع قطع‌کننده مدار (نوسان، اتصال، نرخ خطا) |
| `RiskManager` | محدودیت ضرر روزانه، حجم پوزیشن، slippage |
| `Database` | SQLite با SQLAlchemy async — ذخیره تمام سفارشات |
| `DataCollector` | ثبت معاملات برای آموزش AI در آینده |
| `FastAPI Backend` | رابط REST برای کنترل ربات |

### نقاط مهم تصمیم‌گیری
- سفارش‌ها همیشه از نوع limit هستند (نه market)
- اگر یک سفارش پر شود و دیگری نه → سفارش پر شده کنسل می‌شود
- تمام سفارشات در database ثبت می‌شوند (audit trail)
